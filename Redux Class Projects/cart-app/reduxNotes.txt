# REDUX THUNK FLOW -

Products Component (useEffect)
|
|
dispatch(fetchProducts()) 
|
|
Thunk function runs
|
| --> dispatch(products/fetch/pending)
|      |
|      Store calls the extraReducers 
|      |
|      builder.addCase(fetchProducts.pending) ---> status = "loading"
|
API call fetch happens...
|
|
|
API SUCCESS --> dispatch(products/fetch/fulfilled)
|               |
|               Store calls the extraReducers 
|               |
|               builder.addCase(fetchProducts.fulfilled) ---> data = API DATA, status = "success"
API ERROR -->   dispatch(products/fetch/rejected)
                |
                Store calls the extraReducers 
                |
                builder.addCase(fetchProducts.rejected) ---> status = "failure"


Summary - 
* Thunk acts as middleman / middleware
* It first fires `pending`, then API call, then either `fulfilled` or `rejected`. 
* `extraReducers` listen for those and update the state accordingly. 
* Finally components read the updated state from the Global store with `useSelector()` and render the UI.
